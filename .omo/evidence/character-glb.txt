## Key Implementation Details
- GLTFLoader loads /models/character.glb once (cached in gltfCache); subsequent characters use SkeletonUtils.clone()
- Bounding box computed from mesh geometry only (NOT Box3.setFromObject which includes bone positions — 66x larger)
- computeMeshBounds() helper: traverses meshes, uses Box3.setFromBufferAttribute + applyMatrix4
- Materials swapped to createToonMaterial(color) — single material per character
- AnimationMixer per character; clips shared from cache (AnimationClip is just data)
- Animation mapping: idle→Idle, walk→Walk, run→Run, jump→Jump, talk→Idle (fallback)
- setAnimation stores desired state; plays immediately if mixer ready, else deferred to loadModel
- updateCharacter: dt = time - lastTime (first frame dt=0), clamped to MAX_DT=0.1 (prevents post-pause jumps)
- Fallback: simple primitive humanoid (capsule+icosahedron) if GLB fails to load

## Gotchas
- **Box3.setFromObject includes bone positions for skinned meshes** — the skeleton's bind pose spans ~364 units while the actual mesh geometry is only ~5.5 units tall (mesh local scale 69.18 × geometry 0.08). Using setFromObject for scale calculation produced a character 66x too small (0.03 units). Fix: computeMeshBounds() uses Box3.setFromBufferAttribute on mesh geometry only, transformed by matrixWorld.
- **SkeletonUtils exports named functions, not a namespace** — `import { SkeletonUtils }` fails; must use `import { clone as cloneSkeleton }`.
- **Animation clip names have "Human Armature|" prefix** — matched via case-insensitive includes('idle'/'walk'/'run'/'jump').
- **Model is single mesh with single material** — can't separate clothing/skin/hair by mesh name. NPCs are uniformly colored (spec allows: "或全部服装统一色").
- **dt clamping needed** — when game is paused (hud.isPaused), clock.elapsedTime keeps advancing but updateCharacter isn't called. On resume, dt = time - lastTime would be huge. Clamped to MAX_DT=0.1.
