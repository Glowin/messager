## API Summary
- createHud(questManager): Hud
  - show(): mounts HUD overlay to #app (menu btn ☰ + sound btn ♪ + objective text div); registers Esc keydown listener
  - hide(): removes HUD, closes pause menu if open, removes keydown listener
  - update(): per-frame — reads questManager.getQuests(), finds active quest's current step objectiveText, displays in top-right small text (cached, only updates DOM on change)
  - isPaused (getter): true when pause menu is open

## Design Decisions
- Styles JS-injected (ensureStyles + HUD_CSS const) — tsx compatible, no CSS import
- Design tokens reused via var(--ui-font/--ui-blue/--ui-bg/etc) with fallbacks
- Dark semi-transparent buttons (rgba(15,23,42,0.72) + backdrop-filter blur) — readable against any game background
- Mute state: red diagonal line via ::after on .hud-btn--muted
- Pause menu: z-index 120 (above dialogue z-100), white card + blue PAUSED title + resume/quit buttons
- Quit to title: window.location.reload() (no main.ts modification needed)
- pointer-events: none on root, auto on buttons (non-blocking HUD)
- QuestManagerLike structural interface (accepts QuestManager instance or {getQuests:()=>[]} test doubles)
