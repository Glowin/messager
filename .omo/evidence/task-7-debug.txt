--- Design Notes ---

- import.meta.env.DEV is Vite's static boolean: true in dev, false in prod.
  In PROD, `if (!import.meta.env.DEV) return;` becomes `if (!false) return;`
  → unconditional return → `window.__game = {...}` is dead code → Rollup eliminates it.
- Since no module imports debug-api.ts yet (Wave 4 integration), the ENTIRE
  module is tree-shaken in PROD — zero trace in dist/.
- declare global { interface Window { __game?: DebugApi } } is type-only,
  erased at emit (isolatedModules: true).
- GameState interface: all fields optional (game may not be fully initialized
  when debug API is first installed). Getters return null/[] for missing
  fields — never throw.
- FPS: tickFps() counts frames via performance.now(), recomputes every ~500ms.
  Exported separately so the render loop (main.ts, Wave 4) calls it each frame.
- completeQuest has "仅 QA 用" comment — bypasses quest validation for test scenarios.
- Did NOT touch main.ts/scene.ts/other task files (G17 preserveDrawingBuffer
  lock preserved; integration deferred to Wave 4).
